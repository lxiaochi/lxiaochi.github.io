JSP的概述&入门案例

全称是Java Server pages,他和Servlet一样，也是Java所提供的一门用于开发动态WEB资源的技术，JSP也是JavaEE的规范之一，JSP的底层其实就是Servlet。
JSP的特点

编写JSP就像编写HTML一样
JSP和HTML的而区别
HTML只能向客户端提供静态数据
JSP技术允许我们在页面中嵌套Java代码，从而可以向客户端提供动态的数据

JSP的原理

一个JSP文件第一次被客户端访问时(也包括JSP文件修改后的第一次被访问)，WEB服务器会把JSP文件翻译成一个Servlet，客户端访问JSP文件，其实就是访问JSP文件翻译后的Servlet，JSP的底层原理其实就是Servlet。

JSP默认编码设置

模板元素&脚本片段

模板元素
JSP页面中的HTML标签，css样式，以及JS都称之为JSP的模板元素，模板元素定义了页面的结构和外观，JSP翻译后的Servlet中，模板元素会被输出流对象直接发送到客户端。
脚本片段
JSP页面中可以签到Java代码，Java代码要定义在JSP的脚本片段中，也就是<%%>里面，
JSP页面中可以有多个脚本片段，
多个脚本片段之间，可以嵌套文本，嵌套HTML的标签，嵌套其他的JSP元素。

声明

在JSP页面里定义方法，定义成员变量等，<%! %>

脚本表达式

在JSP页面中将数据发送到客户端,替代JSP页面里的输出语句,
格式：<%=变量/常量/表达式 %>//1+1
注意：在脚本表达式中不可以写分号

注释

HTML的注释：<!-- -->
Java代码的注释：//,/**/
脚本片段的注释：<%-- --%>
脚本片段的注释不会翻译到Servlet

指令的概述

指令并不会产生任何可见的输出，作用是告诉WEB服务器入户处理JSP页面的其他部分。
page
include
taglib
基本格式：
<%@ 指令名称 属性=”值” %>
<%@ page contentType=”text/html;charset=UTF-8” %>
如果一个指令有多个属性，可以把多个属性卸载同一个指令中，也可以吧多个属性分开来写
<%@ page contentType = “text/html;charset=UTF-8” import=”java.util.Date”%>
<%@ page contentType = “text/html;charset=UTF-8”%>
<%@ page import=”java.util.Date”%>

page指令的属性

用于定义JSP页面的各种属性，无论定义在JSP页面的哪个位置，所影响的都是整个JSP页面。
建议将page指令定义在JSP页面的起始位置
属性：
contentType：设置响应的类型和编码
pageEcoding：设置JSP页面的编码，也设置JSP页面翻译成Servlet源文件的编码
import：导入JSP页面所需要使用的Java类
默认已经导入的类
Java.lang.;
Javax.servlet.;
javax.servlet.Jsp.;
javax.servlet.http.;

page指令的其他属性

session：设置是否自动创建内置对象session，默认是true
buffer: 设置内置对象out的缓冲区大小，默认值是8K
autoFlush：设置内置对象out是否会自动刷新，默认是true
extends：设置JSP翻译后的Servlet要继承哪个类，默认值时HttpJspBase
language：设置脚本语言，只有一个值就是java，java也是默认值
isThreadSafe：设置JSP是否是多线程，默认值时true

JSP的异常处理机制

在可能出现错误的JSP页面设置errorPage属性，属性值为URL

include指令&taglib指令

include：用于在JSP页面中包含其他的WEB资源，把多个WEB资源一起翻译到了一个Servlet中
属性：
file：设置要包含的WEB资源的路径
taglib：引入标签库
属性：
url：设置引入标签库的路径
prefix：设置引入标签库的别名

JSP的九大内置对象

  1. HttpServletRequest requset 请求对象
  2. HttpServletResponse response 响应对象
  3. JspWrite out 输出流对象
  4. HttpSession session 会话对象
  5. Throwable exception 异常对象
  6. ServletContext application 代表了整个WEB应用
  7. ServletConfig config 配置参数，用户获取Servlet的配置参数
  8. Object page Servlet的对象，方法总的this对象
  9. PageContext pageContext JSP页面的上下文对象

    PageContext的概述和测试

    JSP技术中最重要的一个对象，呀代表了JSP页面的运行环境
    封装了其他八个内置对象
    也是一个域对象，可以用来保存数据，范围是当前页面，
    提供了一些WEB开发的常用功能，比如转发和包含，还能操作检索其他域对象中的数据。

JSP的四大域对象

  1. pageContext：当前页面 其他页面访问得不到数据
  2. requset：当前请求 不与其他页面共享数据，需要使用转发或者包含
  3. session：整个会话 在整个会话过程中与其他页面共享数据，不同浏览器之间不能共享
  4. application：整个应用 在整个WEB应用实现数据共享，不同浏览器之间也能共享
