jQuery之文档处理

发表于 2018-04-07 | 分类于 JQuery
字数统计: 673 | 阅读时长 ≈ 3

文档处理

append(content|fn)

向每个匹配的元素内部追加内容。
这个操作与对指定的元素执行appendChild方法，将它们添加到文档中的情况类似。

A添加到B : B.append(A)
A可以是JQ对象,js对象,字符串,函数

appendTo(content)

把所有匹配的元素追加到另一个指定的元素元素集合中。

A添加到B : A.appendTo(B)
A ==> JQ对象
B ==> JQ,JS
只有JQ对象才能链式操作JQ方法

prepend(content)

向每个匹配的元素内部前置内容。
这是向所有匹配元素内部的开始处插入内容的最佳方式。

A添加到B : B.prepend(A)
A可以是JQ对象,js对象,字符串,函数

prependTo(content)

把所有匹配的元素前置到另一个、指定的元素元素集合中。

与appendTo(content)相似

after(content|fn)

在每个匹配的元素之后插入内容。

before(content|fn)

在每个匹配的元素之前插入内容。

empty()

删除匹配的元素集合中所有的子节点。
类似于 js 中的 innerHTML = “”

remove([expr])

从DOM中删除所有匹配的元素。
这个方法不会把匹配的元素从jQuery对象中删除，因而可以在将来再使用这些匹配的元素。但除了这个元素本身得以保留之外，其他的比如绑定的事件，附加的数据等都会被移除。

detach([expr])

从DOM中删除所有匹配的元素。
这个方法不会把匹配的元素从jQuery对象中删除，因而可以在将来再使用这些匹配的元素。与remove()不同的是，所有绑定的事件、附加的数据等都会保留下来。

clone([Even[,deepEven]])

克隆匹配的DOM元素并且选中这些克隆的副本。
在想把DOM文档中元素的副本添加到其他位置时这个函数非常有用。

1:一个布尔值（true 或者 false）指示事件处理函数是否会被复制,默认为false。
2:一个布尔值，指示是否对事件处理程序和克隆的元素的所有子元素的数据应该被复制。
